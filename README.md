# コンソールRPG（C++）

## 概要
C++で作成したコンソールベースのRPGゲームです。  
マップ移動、ランダムエンカウント、戦闘システム、パッシブスキル獲得・強化、セーブ・ロードなどを実装しています。

---

## ゲーム内容
- マップ上を移動しながら探索
- ランダムマスで敵とエンカウント
- ランダムにプレイヤーと敵のステータスを変動させる
- 戦闘中に攻撃・防御を選択
- 戦闘終了後に弱パッシブスキル、特定のマスで強パッシブスキルを獲得
- ボスを倒すとゲームクリア
- オートセーブ・ロード

---

## 実装済み機能

### マップ関連
- 二次元配列によるマップ管理
- プレイヤー移動処理
- イベントマス判定  
  - ランダムマス  
  - 回復マス  
  - スキル獲得マス  
  - ボスマス

### 戦闘システム
- ターン制バトル
- 攻撃 / 防御コマンド
- 防御時の被ダメージ軽減
- 敵・プレイヤーのHP管理
- 敵のランダム出現

### キャラクター設計
- `Character` クラスを基底とした  
  - `Player`
  - `Enemy`
- 継承による共通処理の再利用
- 仮想関数によるダメージ処理の分離
- プレイヤーはゲームごとに、敵は戦闘ごとにステータスをランダム変動させる

### ランダムステータス変動
- ゲーム開始時に(データをロードした場合を除く)、プレイヤーのステータスをランダムに変動させる
- エンカウント時に、敵のステータスをランダムに変動させる

### スキルシステム
- `Skill` 抽象クラス（純粋仮想関数 `apply`）
- スキルを `vector<Skill*>` で管理
- スキル獲得時の重複判定とレベルアップ
- 戦闘中に発動する条件付きパッシブスキル

### セーブ仕様
- マップ移動時のコマンド選択ごとに、自動でセーブデータを上書き保存します
- セーブファイルは実行ファイルと同じディレクトリに保存されます

### ロード仕様
- ゲーム起動時、セーブデータが存在する場合は自動でロードされます
- セーブデータが存在しない場合のみ、プレイヤーの初期ステータスがランダム生成されます

### セーブされる内容
- プレイヤーの現在ステータス（HP / 最大HP / ATK / DEF）
- 獲得済みスキルおよびスキルレベル
- マップ上のプレイヤー位置
- 通過済みマスを含むマップ状態

### セーブデータ削除
- ゲームオーバー、ゲームクリア（ボス撃破）時に、オートセーブデータを削除

### 使用技術
- C++
- STL（`vector`）
- 乱数生成（`<random>`）
- クラス設計 / 継承 / 仮想関数
- 状態管理（マップ / 戦闘 / イベント）
- ファイル入出力（`<fstream>` を用いたオートセーブ / ロード）
- ファイル管理（`<filesystem>` によるセーブデータ削除）

## ビルド・実行方法

### 推奨環境
- Windows
- Visual Studio（C++ 開発環境）

### 手順
1. Visual Studioで新規のC++コンソールプロジェクトを作成
2. 本リポジトリ内の `.cpp` / `.h` ファイルをすべてプロジェクトに追加
3. ビルドして実行

※ Visual Studioの自動生成ファイル（.vs / Debug / Release など）は含めていません。

## 工夫した点
- 処理ごとにクラスを分離し、役割を明確化
- スキル処理を仮想関数で統一し、拡張しやすい構成にした
- 戦闘処理とマップ処理を分離して可読性を意識
- 理解しやすい構造を目指した
